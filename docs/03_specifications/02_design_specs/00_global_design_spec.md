# 1. 全局设计规范

本规范是 AutoPPT Agent 设计系统的顶层准则，适用于所有 `layout_type` 的页面。其目的是确保所有生成的演示文稿在信息结构、视觉表现和用户体验上都达到专业、一致的水准。所有页面级的设计规范都是本规范的延伸和具体化。

## 1.1 核心设计原则

- **清晰性（Clarity）**: 设计的首要任务是清晰地传达信息。避免不必要的装饰，确保内容易于阅读和理解。
- **一致性（Consistency）**: 在整个演示文稿中，相同层级、相同类型的元素应具有相同的视觉表现。这有助于降低用户的认知负荷。
- **简洁性（Simplicity）**: “少即是多”。优先使用简洁的布局和有限的色彩，通过留白来突出核心内容。
- **目的性（Purposeful）**: 每一个设计决策都应服务于内容的表达。版式、颜色、字体的选择都应有其明确的理由。

## 1.2 信息层级规范（强制）

清晰的信息层级是专业演示文稿的基石。所有页面都必须严格遵守以下层级规范：

- **层级深度**: 每页最多只允许存在三级标题（H1, H2, H3）。如果内容逻辑超过三级，**必须**通过拆分页面来解决，严禁在单页内创造 H4 或更深层级。
- **层级数量**:
  - `H1` (主标题): 每页最多 **1** 个。
  - `H2` (副标题/章节标题): 每页建议不超过 **2** 个。
  - `H3` (子标题/要点标题): 每页建议不超过 **4** 个。

## 1.3 布局与网格规范

统一的布局和网格系统是保证视觉一致性的关键。

- **画布比例**: 所有页面的宽高比固定为 **16:9**。
- **安全边距**: 为避免内容过于贴近边缘，所有页面元素都应在安全边距内部。推荐设置为：
  - 左右边距：`0.35英寸`
  - 上下边距：`0.25英寸`
- **垂直间距**: 元素间的垂直间距应保持一致的节奏感。推荐：
  - 标题区与正文区之间：至少 `0.18英寸`

## 1.4 文本内容规范

- **标题长度**: `H1` 主标题建议不超过 **18** 个中文字符（或 12 个英文单词），以保证其视觉冲击力。
- **段落长度**: 每个正文段落建议不超过 **3** 行，以提高可读性。
- **列表项**: 列表（bullet points）的条目数建议在 **3-7** 条之间。单条内容不应过长，建议不超过 **24** 个中文字符。
- **内容溢出处理**: 如果文本内容过长，必须按以下顺序进行处理：**1. 缩写或精简 -> 2. 拆分段落 -> 3. 拆分页面**。

## 1.5 禁用项

- **禁止混用版式**: 单个页面不应混合多种复杂的版式语义，例如，在一个时间轴页面中再插入一个复杂的对比表格。
- **禁止过度动画**: 作为信息承载工具，应避免使用过多或过于花哨的动画效果，以免分散观众注意力。

本全局规范的详细实现，请参考字体、配色和布局网格等专项规范文档。
