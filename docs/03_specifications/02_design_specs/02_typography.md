# 3. 字体排版规范

字体和排版是构建清晰信息层级和保证内容可读性的基础。本规范定义了 AutoPPT Agent 生成演示文稿时所遵循的字体选择、字号体系和排版细节，以确保视觉的专业性和一致性。

## 3.1 字体选择

为了保证在不同操作系统和设备上的兼容性和显示效果，我们选择广泛预装的、高质量的系统字体作为默认字体族。

- **中文主字体**: **Microsoft YaHei (微软雅黑)**
  - 理由：作为 Windows 和 Office 套件的默认中文字体，具有极高的普及率。其字形现代、清晰，非常适合屏幕显示。

- **英文及数字字体**: **Calibri**
  - 理由：同样是 Office 的默认西文字体，与微软雅黑搭配协调。其人性化的设计和清晰的字形使其在各种字号下都表现出色。

- **备选方案**: 在无法使用上述字体的环境中（如某些 Linux 发行版），系统应优雅地降级到其他高质量的无衬线字体，如 `PingFang SC` (macOS) 或 `Noto Sans CJK`。

## 3.2 字号体系

我们建立了一个基于层级的、有节奏感的字号体系，以明确区分不同级别的信息。

| 信息层级 | 角色 | 建议字号范围 (pt) | 字重建议 |
| :--- | :--- | :--- | :--- |
| H1 | 页面主标题 | 34 - 42 | 加粗 (Bold) |
| H2 | 副标题 / 章节标题 | 24 - 30 | 半粗 (Semi-bold) |
| H3 | 子标题 / 要点标题 | 18 - 22 | 常规 (Regular) 或 半粗 |
| Body | 正文 / 列表项 | 16 - 20 | 常规 (Regular) |
| Caption | 注释 / 来源 / 图例 | 11 - 13 | 常规 (Regular) |

**使用原则**:

- **拉开差距**: 不同层级之间的字号应有足够大的差距，以在视觉上清晰地区分它们。
- **保持克制**: 在单个页面中，建议使用的字重种类不超过 **2** 种（例如，常规 + 加粗），以避免视觉噪声。

## 3.3 排版细节

- **行高 (Line Spacing)**: 为了保证大段文本的可读性，建议行高设置为字号的 **1.3 - 1.5** 倍。

- **段间距 (Paragraph Spacing)**: 段落之间应留有足够的呼吸空间，建议设置 **0.5 - 1** 倍的行高作为段后间距，而不是使用空行来分隔段落。

- **对齐方式**: 
  - 标题：通常采用左对齐或居中对齐。
  - 正文：强制使用左对齐，以保证段落边缘的整齐和阅读的流畅性。**禁止**使用两端对齐，因为它会在文本中产生不自然的空白间隙。

## 3.4 渲染引擎要求

渲染引擎在处理文本时，必须能够根据 `python-pptx` 库的接口，精确地设置文本框中每个段落的字体、字号、字重、颜色、行高和对齐方式，以严格执行本规范。
