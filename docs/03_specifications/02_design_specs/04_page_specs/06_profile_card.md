# `profile_card` 设计规范

## 1. 用途

`profile_card` 版式用于在页面中突出介绍一个核心人物，例如团队领导、项目负责人、特邀专家或关键贡献者。它旨在以一种视觉化、结构化的方式，快速传递人物的核心信息。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题，例如“项目负责人”或“核心团队成员”。
- `profiles`: 一个**只包含一个对象**的数组，该对象描述了人物信息。

### 结构定义

`profiles` 数组中的对象必须包含以下字段：

```typescript
interface Profile {
  name: string;                // 人物姓名
  role: string;                // 角色或职位
  bio: string;                 // 个人简介
  avatar_local_path?: string;  // (可选) 指向本地头像图片的路径
  avatar_search_query?: string;// (可选) 用于搜索头像的查询词
}
```

### 示例

```json
{
  "page_id": 6,
  "layout_type": "profile_card",
  "title": "项目首席科学家",
  "profiles": [
    {
      "name": "李教授",
      "role": "人工智能领域专家",
      "bio": "在自然语言处理和计算机视觉领域拥有超过15年的研究经验，发表了多篇顶级会议论文，并领导了多个国家级AI项目。",
      "avatar_search_query": "professional portrait of a senior male asian scientist"
    }
  ]
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `Profile`
- **建议母版索引**: `6`
- **必需占位符**:
  - `title`: 用于页面主标题。
  - `name`: 用于人物姓名。
  - `role`: 用于人物角色或职位。
  - `bio`: 用于个人简介文本。
  - `avatar`: 用于展示人物头像的图片占位符。

## 4. 设计与排版规则

- **信息层级**: `name` 应作为页面的视觉焦点之一，其字号和字重应显著高于 `role` 和 `bio`。
- **文本长度**: `bio` 的文本长度建议在 **40-120** 字之间，以提供足够的信息，同时避免页面显得过于拥挤。
- **头像处理**: 头像图片建议使用 **1:1** 或 **4:5** 的宽高比。渲染引擎在填充 `avatar` 占位符时，必须使用 `Crop-to-Fill` 算法，确保头像不变形。

## 5. 禁用项

- **禁止展示多人**: 此版式严格用于介绍**单个人物**。如果需要同时展示多个成员，**必须**使用 `team_grid` 版式。

## 6. 验收清单

- [ ] `profiles` 数组中恰好只包含一个对象。
- [ ] 人物的 `name`, `role`, `bio` 字段内容完整。
- [ ] 头像图片已正确填充，且无任何拉伸或变形。
- [ ] 最终生成的 `.pptx` 文件中，所有文本内容均可编辑。
