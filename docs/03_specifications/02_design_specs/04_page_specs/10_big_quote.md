# `big_quote` 设计规范

## 1. 用途

`big_quote` 版式是一种高度风格化的页面，专门用于突出和强调一个核心观点、一句有力的引言、或一段结论性的陈述。它的目标是通过强烈的视觉设计，将观众的注意力完全集中在这句话上，给观众留下深刻印象。它常被用作章节的过渡页或演示的总结页。

## 2. 数据协议

### 必需字段

- `quote`: 要突出显示的引言或核心观点文本。

### 可选字段

- `author`: 引言的出处或作者。如果提供，会以较小的字号显示在引言下方。
- `title`: 页面的主标题，例如“核心结论”或“专家观点”。

### 示例

```json
{
  "page_id": 10,
  "layout_type": "big_quote",
  "title": "我们的愿景",
  "quote": "我们相信，技术应赋能创造，而非取代思考。",
  "author": "公司创始人"
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `Quote`
- **建议母版索引**: `10`
- **必需占位符**:
  - `quote`: 用于引言文本的主占位符。
  - `author`: (可选) 用于作者信息的占位符。
  - `title`: (可选) 用于页面标题的占位符。

## 4. 设计与排版规则

- **视觉焦点**: `quote` 文本是页面的绝对视觉中心。它的字号应非常大（通常在 **36-52pt** 范围），并使用加粗或特殊的艺术字体（如果母版支持）来增强其视觉冲击力。
- **大量留白**: 页面应保持极度简洁，引言周围必须有大量的留白，以突出其重要性。
- **文本长度**: `quote` 的文本长度应严格控制，建议不超过 **30** 个中文字符（或20个英文单词），以保证其能够以大字号完整显示。
- **作者信息**: 如果提供了 `author`，其字号应远小于 `quote`，并通常放置在引言的右下方，作为补充说明。

## 5. 禁用项

- **禁止承载复杂信息**: 严禁在此版式中添加列表、表格、长段落或任何其他复杂信息。页面的唯一目的就是突出引言本身。

## 6. 验收清单

- [ ] `quote` 字段内容完整且长度适中。
- [ ] 引言文本在页面上居于视觉中心，并有足够的留白。
- [ ] 页面整体设计简洁，无不相关的干扰元素。
- [ ] 最终生成的 `.pptx` 文件中，引言和作者文本均可编辑。
