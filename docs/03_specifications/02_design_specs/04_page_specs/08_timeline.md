# `timeline` 设计规范

## 1. 用途

`timeline` 版式专门用于按时间顺序展示一系列事件、里程碑或发展阶段。它非常适用于介绍公司发展史、项目路线图（Roadmap）、产品迭代历程或一个事件的演变过程。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题，例如“发展历程”或“项目路线图”。
- `timeline_items`: 一个包含 **3-5 个对象**的数组，每个对象代表时间轴上的一个节点。

### 结构定义

`timeline_items` 数组中的每个对象必须包含以下字段：

```typescript
interface TimelineItem {
  year: string;   // 节点的时间标识（可以是年份、季度或具体日期）
  title: string;  // 该节点的事件标题
  desc: string;   // 对事件的简要描述
}
```

### 示例

```json
{
  "page_id": 8,
  "layout_type": "timeline",
  "title": "公司发展里程碑",
  "timeline_items": [
    { "year": "2021", "title": "公司成立", "desc": "获得天使轮融资，组建核心创始团队。" },
    { "year": "2022", "title": "产品上线", "desc": "发布首个MVP版本，积累早期用户。" },
    { "year": "2024", "title": "市场扩张", "desc": "用户数突破100万，完成A轮融资。" },
    { "year": "2026", "title": "生态构建", "desc": "计划开放平台API，构建开发者生态。" }
  ]
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `Timeline`
- **建议母版索引**: `8`
- **必需占位符**: 母版中应包含用于构建时间轴的多个占位符组，例如对于一个最多支持5个节点的母版：
  - `title`: 用于页面主标题。
  - `year_1`, `title_1`, `desc_1`
  - `year_2`, `title_2`, `desc_2`
  - ...
  - `year_5`, `title_5`, `desc_5`

## 4. 设计与排版规则

- **节点数量**: 为了保证页面的清晰度和可读性，时间轴上的节点数量建议为 **3-5** 个。如果事件过多，应筛选出最重要的里程碑，或将其拆分为多个页面。
- **时间顺序**: `timeline_items` 数组中的元素**必须**严格按照时间先后顺序排列。
- **文本长度**: 每个节点的描述文本（`desc`）建议不超过 **40** 个中文字符，保持信息的简洁性。
- **视觉连接**: 母版设计中应包含清晰的视觉元素（如线条、箭头）来连接各个时间节点，以明确地指示时间的流向。

## 5. 禁用项

- **禁止非时序内容**: 此版式严格用于展示与时间相关的内容，禁止用于罗列无时间顺序的要点。

## 6. 验收清单

- [ ] `timeline_items` 数组中的对象数量在 3-5 个之间。
- [ ] 数组中的元素已按时间顺序排列。
- [ ] 每个节点的 `year`, `title`, `desc` 字段内容完整。
- [ ] 页面布局清晰，时间轴的视觉引导性强。
