# `agenda_list` 设计规范

## 1. 用途

`agenda_list` 版式用于在演示文稿的早期阶段（通常在封面页之后）向观众展示内容的整体结构。它适用于创建目录、会议议程、或者对即将讨论的核心要点进行总览。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题，例如“目录”或“主要议程”。
- `bullet_points`: 一个字符串数组，包含了要逐条列出的所有要点。

### 示例

```json
{
  "page_id": 2,
  "layout_type": "agenda_list",
  "title": "今日议程",
  "bullet_points": [
    "第一季度业绩回顾",
    "市场竞争格局分析",
    "下季度战略规划",
    "开放讨论与问答"
  ]
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `Agenda`
- **建议母版索引**: `1`
- **必需占位符**:
  - `title`: 用于页面主标题。
  - `content`: 一个文本框占位符，渲染引擎将在此处循环创建列表项。

## 4. 设计与排版规则

- **列表项数量**: 为了保持页面的简洁性和可读性，`bullet_points` 的数量建议在 **3-7** 条之间。如果超过7条，应考虑将其拆分为多个页面或进行更高层次的概括。
- **列表项长度**: 每个列表项的文本长度建议不超过 **20** 个中文字符，以避免单行文本过长或需要换行。
- **列表层级**: 建议只使用单层列表，避免出现多级缩进的复杂结构，以保持目录的清晰性。
- **间距**: 列表项之间应保持统一的垂直间距，以创造稳定、有序的视觉节奏。

## 5. 禁用项

- **禁止长段落**: 此版式不适用于展示大段的详细描述性文字。
- **禁止混合结构**: 严禁在此页面中混入表格、图表或时间轴等其他复杂结构。

## 6. 验收清单

- [ ] `title` 和 `bullet_points` 字段内容完整。
- [ ] 列表项数量在合理的 3-7 条范围内。
- [ ] 页面布局清晰，文本无明显溢出。
- [ ] 列表项对齐整齐，间距一致。
