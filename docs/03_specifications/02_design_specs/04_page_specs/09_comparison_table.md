# `comparison_table` 设计规范

## 1. 用途

`comparison_table` 版式专门用于清晰地对比两个方案、产品、策略或概念的优劣势。它通过并列的表格形式，帮助观众快速理解两者之间的关键差异。此版式强调的是**对抗性**或**选择性**的比较。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题，例如“方案 A vs 方案 B”。
- `comparison_items`: 一个包含 `left` 和 `right` 两个属性的对象，分别描述对比双方的内容。

### 结构定义

`comparison_items` 对象及其子对象的结构如下：

```typescript
interface ComparisonItems {
  left: ComparisonSide;
  right: ComparisonSide;
}

interface ComparisonSide {
  header: string;   // 对比方的名称，如“方案 A”
  items: string[];  // 一个包含该方特点或优势的字符串数组
}
```

### 示例

```json
{
  "page_id": 9,
  "layout_type": "comparison_table",
  "title": "自研 vs 采购 决策对比",
  "comparison_items": {
    "left": {
      "header": "自研开发",
      "items": [
        "+ 高度定制化，完全贴合业务",
        "+ 掌握核心技术，数据安全可控",
        "- 开发周期长，初始投入成本高"
      ]
    },
    "right": {
      "header": "采购成熟方案",
      "items": [
        "+ 部署速度快，可快速上线",
        "+ 经过市场验证，系统稳定",
        "- 定制化能力弱，难以满足特殊需求"
      ]
    }
  }
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `Comparison`
- **建议母版索引**: `9`
- **必需占位符**:
  - `title`: 用于页面主标题。
  - `header_l`, `header_r`: 分别用于左右两侧的标题。
  - `content_l`, `content_r`: 分别用于左右两侧的要点列表。

## 4. 设计与排版规则

- **明确对比关系**: 强烈建议在 `items` 的文本前使用 `+`、`-` 或 `✔`、`✘` 等符号，以明确表示该项是优势还是劣势，增强对比的可读性。
- **维度对齐**: 如果可能，左右两侧的 `items` 应尽量在同一维度上进行对比，并保持一一对应。例如，如果左侧第一项是“成本”，右侧第一项也应该是“成本”。
- **要点数量**: 每侧的要点数量建议为 **2-4** 条，以突出最关键的差异点。
- **视觉分隔**: 母版设计中应包含清晰的视觉分隔线，以在空间上明确区分对比的双方。

## 5. 禁用项

- **禁止非对比内容**: 此版式严格用于具有明确对比关系的内容。对于只是并列展示、无强烈对比意味的内容，应使用 `section_two_col` 版式。

## 6. 验收清单

- [ ] `comparison_items` 对象及其 `left` 和 `right` 属性内容完整。
- [ ] 对比项使用了 `+`/`-` 等符号来明确优劣关系。
- [ ] 左右两侧内容在视觉上保持了良好的对齐和平衡。
- [ ] 最终生成的 `.pptx` 文件中，所有文本内容均可编辑。
