# `section_four_grid` 设计规范

## 1. 用途

`section_four_grid` 版式提供了一个结构化的 2x2 网格布局，专门用于需要从四个维度进行分析或展示信息的场景。最典型的应用是 **SWOT 分析（优势、劣势、机会、威胁）**，但它同样适用于任何可以归纳为四个对等象限的分析模型。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题，例如“SWOT 分析”。
- `grid_items`: 一个包含四个对象的数组，每个对象代表一个网格单元。

### 结构定义

`grid_items` 数组中的每个对象必须包含以下字段：

```typescript
interface GridItem {
  title: string; // 网格单元的标题，如“优势 (Strengths)”
  desc: string;  // 对该维度的简要描述
}
```

### 示例

```json
{
  "page_id": 5,
  "layout_type": "section_four_grid",
  "title": "项目竞争力 SWOT 分析",
  "grid_items": [
    { "title": "优势 (S)", "desc": "拥有自主研发的核心AI算法和完整的技术团队。" },
    { "title": "劣势 (W)", "desc": "品牌知名度较低，市场推广资源有限。" },
    { "title": "机会 (O)", "desc": "企业数字化转型带来对自动化内容生成工具的巨大需求。" },
    { "title": "威胁 (T)", "desc": "大型科技公司可能进入该领域，带来激烈竞争。" }
  ]
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `4_Grid`
- **建议母版索引**: `5`
- **必需占位符**:
  - `title`: 用于页面主标题。
  - `text_1`, `text_2`, `text_3`, `text_4`: 四个独立的文本框占位符，分别对应 `grid_items` 数组中的四个元素。

## 4. 设计与排版规则

- **内容均衡**: 四个网格单元中的信息量应力求均衡，避免出现某个单元内容过多而其他单元内容过少的情况，以维持视觉上的稳定。
- **文本长度**: 每个单元的描述文本（`desc`）建议在 **20-50** 字之间，确保信息简洁明了。
- **视觉权重**: 四个单元格在设计上应具有同等的视觉权重，不应特别突出或放大其中任何一个。
- **对齐**: 四个单元格应在网格中严格对齐，包括其内部标题和描述文本的对齐方式。

## 5. 禁用项

- **禁止变动数量**: 此版式严格用于展示 **四个** 主分项，禁止用于多于或少于四个项目的情况。

## 6. 验收清单

- [ ] `title` 和 `grid_items` 字段内容完整，且 `grid_items` 数组恰好包含四个元素。
- [ ] 四个网格单元均有内容，且无明显文本溢出或布局错位。
- [ ] 最终生成的 `.pptx` 文件中，所有文本内容均可编辑。
