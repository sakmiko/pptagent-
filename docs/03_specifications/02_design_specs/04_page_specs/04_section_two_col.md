# `section_two_col` 设计规范

## 1. 用途

`section_two_col` 版式专门用于在单个页面上并列介绍或比较两个对等的概念、方案或信息模块。它强调的是并列展示，而非强烈的对抗性对比。例如，可以用它来介绍一个问题的两个不同方面，或者一个策略的两个支撑点。

## 2. 数据协议

### 必需字段

- `title`: 页面的主标题。
- `left_section`: 一个描述左侧栏内容的对象。
- `right_section`: 一个描述右侧栏内容的对象。

### 结构定义

`left_section` 和 `right_section` 对象应包含以下字段：

```typescript
interface Section {
  header: string;   // 栏目标题
  items: string[];  // 一个包含该栏要点的字符串数组
}
```

### 示例

```json
{
  "page_id": 4,
  "layout_type": "section_two_col",
  "title": "增长驱动力拆解",
  "left_section": {
    "header": "技术驱动",
    "items": [
      "电池效率持续提升",
      "高压快充平台普及",
      "智能化座舱体验升级"
    ]
  },
  "right_section": {
    "header": "市场驱动",
    "items": [
      "国家政策大力支持",
      "用户环保意识增强",
      "充电基础设施完善"
    ]
  }
}
```

## 3. 母版与占位符

- **母版 Layout 名称**: `2_Column`
- **建议母版索引**: `4`
- **必需占位符**:
  - `title`: 用于页面主标题。
  - `text_l`: 用于填充左栏内容的文本框。渲染引擎会在此处填入 `left_section` 的 `header` 和 `items`。
  - `text_r`: 用于填充右栏内容的文本框。渲染引擎会在此处填入 `right_section` 的 `header` 和 `items`。

## 4. 设计与排版规则

- **内容均衡**: 左右两栏的文本信息量应大致相当，视觉重量差异不应超过 **30%**，以维持页面的平衡感。
- **要点数量**: 每栏的要点数量建议为 **2-4** 条，以保证信息的简洁和易读性。
- **层级一致**: 两栏的 `header` 和 `items` 在字号、字重和样式上应保持严格一致，不允许出现一侧的视觉强调度高于另一侧的情况。
- **对齐**: 两栏的顶部应严格对齐，创造整洁的视觉秩序。

## 5. 禁用项

- **禁止强对抗内容**: 此版式不适用于展示具有强烈优劣对比或“A vs B”性质的内容。对于这类内容，**必须**使用 `comparison_table` 版式，以更清晰地突出其对抗关系。

## 6. 验收清单

- [ ] `title`, `left_section`, `right_section` 字段内容完整。
- [ ] 左右两栏内容对齐整齐，视觉上保持平衡。
- [ ] 文本内容未溢出各自的文本框。
- [ ] 左右两栏的标题和列表样式完全可编辑。
