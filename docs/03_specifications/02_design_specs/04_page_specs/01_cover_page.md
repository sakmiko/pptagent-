# `cover_page` 设计规范

## 1. 用途

`cover_page` 版式是演示文稿的“第一印象”，专门用于封面页（第1页）。其核心任务是清晰地传达演示文稿的主题，并吸引观众的注意力，同时建立整个演示的视觉基调。

## 2. 数据协议

### 必需字段

为了正确渲染封面页，JSON 数据中必须提供以下字段：

- `title`: 演示文稿的主标题。
- `subtitle`: 对主标题进行补充说明的副标题。

### 可选字段

- `image_local_path` 或 `image_search_query`: 用于背景的图片。前者是已处理好的本地图片路径，后者是用于触发图片搜索的查询词。

## 3. 母版与占位符

- **母版 Layout 名称**: `Cover`
- **建议母版索引**: `0`
- **必需占位符**: 渲染引擎将查找并填充以下占位符：
  - `title`: 用于主标题文本。
  - `subtitle`: 用于副标题文本。
  - `bg_image`: 用于背景图片的图片占位符。

## 4. 设计与排版规则

- **信息层级**: `title` 的字号和字重应显著高于 `subtitle`，形成清晰的主次关系。两者之间的字号差距建议至少为 **8pt**。
- **文本长度**: 
  - `title` 建议不超过 **18** 个中文字符（或12个英文单词），以保证其视觉冲击力。
  - `subtitle` 建议不超过 **28** 个中文字符。
- **背景图处理**: 背景图片应作为视觉氛围的补充，不能干扰标题的可读性。如果图片与文字颜色对比度不足，设计 Agent 应指示渲染引擎自动在图片上添加一个半透明的暗色遮罩（Overlay）以增强对比度。
- **视觉焦点**: 整个页面应保持简洁，视觉焦点应集中在标题上。避免在封面页添加除标题和背景图之外的任何其他分散注意力的元素。

## 5. 禁用项

- **禁止承载密集信息**: 严禁在封面页放置列表、表格、长段落正文或复杂图表。
- **禁止多焦点**: 页面应只有一个核心视觉焦点（即标题）。

## 6. 验收清单

- [ ] 必须用于且仅用于第1页。
- [ ] `title` 和 `subtitle` 字段内容完整。
- [ ] 标题文本清晰可读，与背景有足够的对比度，无明显遮挡。
- [ ] 最终生成的 `.pptx` 文件中，标题和副标题的文本框是可编辑的。
