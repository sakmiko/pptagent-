# 技术栈说明

本文档详细定义了 AutoPPT Agent 项目所采用的全栈技术选型，旨在为开发人员提供一个清晰、统一的技术框架，确保开发效率和项目的可维护性。

## 1. 总体架构

我们采用前后端分离的现代 Web 应用架构，并通过一个异步任务队列来处理计算密集型的 PPT 生成任务。

```mermaid
graph TD
    A[前端 (React + Vite)] --> B{后端 API (FastAPI)};
    B --> C[任务队列 (RabbitMQ)];
    C --> D[PPT 生成 Worker (LangGraph + python-pptx)];
    D --> E[对象存储 (MinIO/S3)];
    B --> E;
    A --> E;
```

## 2. 前端技术栈

| 技术 | 版本 | 用途与说明 |
| :--- | :--- | :--- |
| **React** | 18.x | 核心 UI 框架，采用函数式组件和 Hooks。 |
| **Vite** | 5.x | 下一代前端构建工具，提供极速的开发服务器和优化的打包。 |
| **TypeScript** | 5.x | 为 JavaScript 提供静态类型检查，增强代码的健壮性和可维护性。 |
| **Tailwind CSS** | 3.x | 一个功能优先的 CSS 框架，用于快速构建现代化的、响应式的用户界面。 |
| **Zustand** | 4.x | 一个轻量级、简洁的状态管理库，用于管理全局 UI 状态。 |
| **React Query** | 5.x | 用于管理服务器状态，包括数据获取、缓存、同步和更新。 |
| **Axios** | 1.x | 用于发起 HTTP 请求，与后端 API 进行通信。 |

## 3. 后端技术栈

### 3.1 API 服务

| 技术 | 版本 | 用途与说明 |
| :--- | :--- | :--- |
| **Python** | 3.10+ | 主要的后端编程语言。 |
| **FastAPI** | 0.100+ | 一个现代、高性能的 Python Web 框架，用于构建 API。自带基于 OpenAPI 的交互式文档。 |
| **Pydantic** | 2.x | 用于数据验证和设置管理。FastAPI 深度集成 Pydantic，确保 API 输入输出的类型安全。 |
| **Uvicorn** | 0.20+ | 一个闪电般快速的 ASGI 服务器，用于运行 FastAPI 应用。 |

### 3.2 PPT 生成 Worker

| 技术 | 版本 | 用途与说明 |
| :--- | :--- | :--- |
| **LangGraph** | 0.0.20+ | 一个用于构建有状态、多 Agent 应用的库。我们用它来编排整个八阶段的 PPT 生成流水线。 |
| **LangChain** | 0.1.0+ | 用于与大型语言模型（LLM）进行交互，封装了 Prompt 模板、输出解析器和工具调用等功能。 |
| **python-pptx** | 0.6.20+ | 核心的 PowerPoint 文件生成库，用于创建和编辑 `.pptx` 文件。 |
| **Pillow** | 10.x | 用于在将图片插入 PPT 前进行裁剪、缩放等预处理。 |
| **Celery** | 5.x | 一个强大的分布式任务队列，用于异步执行耗时的 PPT 生成任务。 |
| **RabbitMQ** | 3.x | 作为 Celery 的消息代理（Broker），负责任务的分发和存储。 |

## 4. 数据与存储

| 技术 | 用途与说明 |
| :--- | :--- |
| **PostgreSQL** | 用于存储结构化数据，如用户信息、任务状态、历史记录等。 |
| **MinIO / AWS S3** | 用于对象存储，存放生成的 `.pptx` 文件、用户上传的素材、以及中间过程产生的图片等非结构化数据。 |

## 5. 外部服务依赖

| 服务 | 用途 |
| :--- | :--- |
| **OpenAI API** | 作为核心的大型语言模型（LLM）提供商，驱动所有 Agent 的智能。 |
| **Tavily Search API** | 提供高质量、结构化的网络搜索结果，作为研究阶段的主要信息来源。 |
| **Bing Image Search API** | 用于根据内容自动搜索和下载配图。 |
