# 4. 技术栈

AutoPPT Agent 项目的技术栈选型旨在平衡开发效率、系统性能、AI 能力和可维护性。我们优先选择社区成熟、生态完善且与 AI Agent 开发高度契合的技术和工具。

## 4.1 核心技术栈

下表概述了项目各主要模块推荐使用的核心技术和库。

| 模块 | 推荐技术/库 | 选型理由 |
| :--- | :--- | :--- |
| **编程语言** | **Python 3.10+** | 作为 AI 和数据科学领域的首选语言，拥有最丰富的生态系统和社区支持，能够无缝集成各类 AI 框架和库。 |
| **LLM 框架** | **LangChain / LangGraph** | 提供了强大的工具来构建、管理和编排复杂的 Agent 应用。其状态管理、工具调用和链式组合能力是本项目的核心。 |
| **LLM 模型** | **GPT-4o / Claude 3.5 Sonnet** | 项目对模型的长文本理解、指令遵循和 JSON 输出能力有极高要求。这两款模型是当前市场的领先者，能最好地满足需求。 |
| **搜索引擎** | **Tavily API** | 专为 AI Agent 设计的搜索引擎，返回的结果格式干净、结构化，无需复杂的网页爬取和数据清洗，极大简化了信息获取流程。 |
| **图片搜索** | **Bing Image Search API** | 提供了高质量的图片搜索结果，并支持按许可证类型进行筛选，有助于确保所用素材的合规性。 |
| **PPT 生成** | **python-pptx** | 是 Python 生态中唯一成熟且功能完备的库，用于创建和修改 PowerPoint (`.pptx`) 文件，是实现原生可编辑输出的关键。 |
| **Web UI (MVP)** | **Streamlit** | 在项目初期和原型验证（MVP）阶段，Streamlit 能够以极高的效率构建交互式的前端界面，便于快速调试和演示。 |
| **Web UI (Production)** | **FastAPI + React/Vue** | 在生产环境中，采用 FastAPI 作为高性能后端 API 服务，搭配 React 或 Vue 等现代前端框架，可以构建功能更强大、用户体验更佳的 Web 应用。 |

## 4.2 技术选型原则

- **拥抱开源**: 优先选择活跃的开源项目，以受益于社区的持续贡献和快速迭代。
- **成熟稳定**: 核心依赖库选择经过大规模生产环境验证的版本，避免引入不稳定性。
- **AI 优先**: 在工具选型上，优先考虑那些为 AI 应用（尤其是 Agent）优化过的服务和框架。
- **模块化与解耦**: 技术栈的选择应服务于系统的模块化设计，确保各组件之间低耦合，易于独立升级或替换。

详细的架构设计和组件交互逻辑，请参阅 **[系统架构文档](./03_system_architecture.md)**。
